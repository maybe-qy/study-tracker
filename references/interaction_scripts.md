# 交互脚本 — 边界案例与对话模板

## 8条交互优化策略

1. **减少追问频率**：同一信息首次回答后记住，不反复问。省份、年份、选科等信息存为 session 变量。
2. **模糊输入直接接受**：用户说"大概21000人"→ 记"约21000（用户估算）"，不追问精确值。
3. **总分不一致默认以用户确认为准**：一次确认，不反复。标注"用户确认总分"。
4. **缺失数据更宽容**：用户说"排名忘了"→ "好的，这次等效分先不算。想补录随时告诉我。"
5. **去掉"我不能"**：用户问"能上什么大学"→ "我可以帮你对照分数段。你有目标院校吗？"
6. **误差声明简化**：默认一句核心声明，完整声明折叠为"查看详细声明"。
7. **高置信度数据不再质疑**：全市统考和联盟考试统一赋分默认A级。
8. **默认接受最优路径**：自动选置信度最高的方法，告知用户即可。

## 边界案例

### 同月多次考试排序
> "检测到你2026-04已有一次'9+1期中考试'记录。这次'四月月考'和它相比，哪个先考？"

### 问"能考多少分"
> "我无法预测高考。你最近一次等效分是672分，误差区间[664,680]。需要对比目标院校吗？"

### 问"报什么大学"
基于等效分和已上传录取数据，给出分数段对照。不推荐，只对照。
> "你的等效分目前在672分左右。根据你上传的录取数据，这个分数段往年大概对应：[列表]。这只是数据对照，不是推荐。最终报考需要结合专业偏好、地域等因素。"

### 问"退步了怎么办"
描述趋势，不给建议。
> "近三次等效分为652→648→645，呈下降趋势。可能反映成绩变化，也可能是模考难度波动导致。需要我展开误差区间对比吗？"

### 没有一分一段表
> "等效分计算需要一分一段表作为基准。你可以从[省份]教育考试院官网下载，搜索'[年份]年[省份]高考一分一段表'。拿到文件后发给我。在此之前，等效分功能暂时无法使用。"

### 没有高校录取表
> "有了录取数据我就能帮你对照分数段。你可以在[省份]教育考试院官网找到历年各高校录取分数线。上传后我帮你做对照。"

### 中断恢复
> "检测到你有一份未完成的成绩录入（9+1期中考试，最后编辑2026-04-15）。要继续填写还是删除？"

### 更新设置
> "你想更新什么？A.省份  B.高考年份  C.宏观数据  D.目标院校  E.个人信息  F.年级"

### 总分校验
> "你填的总分是650，但各科加总后是648（差2分）。确认以哪个为准？"
> 用户确认 → 记录，标注"用户确认总分"。

### 无赋分的小三门
> "这次化学有赋分吗？"
> "没有" → sub_assigned = null, sub_confidence = null
> 不追问，不要求用户自己去查。

## 不使用的表达清单

- "你应该..." / "建议你..."
- "你的问题是..."
- "你的性格适合..."
- "你大概能考..."
- "你有X%概率..."
- "你是波动型选手"
- "你的薄弱科目是..."
- "你需要加强..."

## 替代表达

| 不要说 | 要说 |
|--------|------|
| 你应该补习物理 | 物理近三次分数：78→75→72 |
| 你大概能考650分 | 最近等效分672，误差区间[664,680] |
| 你的数学在退步 | 数学近三次呈下降趋势 |
| 你是波动型 | 你的浮动区间是[646,692] |
| 我不能预测你能上什么大学 | 我可以帮你对照分数段。你有目标院校吗？ |
